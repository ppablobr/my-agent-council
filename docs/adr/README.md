# Architecture Decision Records (ADRs)

ADRs capture technical decisions that are hard to reverse or shape the system long-term.

## Index

| ADR | Title | Status |
| --- | --- | --- |
| [0001](./0001-spec-driven-development.md) | Spec-Driven Development Methodology | Accepted |

## When to Write an ADR

- New architecture patterns (routing, state management, data access)
- Major dependency or stack decisions
- Data model and persistence decisions
- Security, privacy, or compliance-related design choices

## Naming

- Use `docs/adr/NNNN-short-title.md` (4-digit sequence).
- Example: `docs/adr/0002-auth-strategy.md`

## Template

Start from `docs/adr/0000-template.md`.
